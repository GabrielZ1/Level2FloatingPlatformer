Floating Platformer

by: Gabriel Oviedo

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted to make a fun and challenging platformer game while using what I've learned about ArrayLists to make the objects.

2. What I Learned

I learned that the game was challenging so I added a keybind ("0") that skips a level if needed.

3. Architecture of My Game

I had 15 total classes in my game with about 4 main classes: the runner class, a game panel, an object manager, and an object class. The other 11 classes are the player and all of the game objects and powerups. The game objects inherit their x, y, width, and height from the "GameObject" class. The player and each object has its own collision box. When the player's collision box and an object's collision box intersect, the object manager checks it and has the appropriate result occur. Each level is designed in the game panel when it calls an "add [object]" method from the object manager to add an object to an ArrayList and draws them.

4. Technical Implementation

Making the collision between the player and the platform was a difficult task to overcome. The lower part of the player on the top of the platform was not too difficult, but preventing the player from going inside of the bottom and sides of the platform was a lot harder. For the sides, a limit of x-movement had to be implemented with a bit of manual adjustment (with trial and error) to prevent gameplay from feeling odd. For the bottom, a limit on the y-movement was necessary to prevent the player from going through them.

5. Most Challenging Part

The hardest part of the project was fixing all of the bugs that appeared whenever I implemented a feature. When I added a double jump, a lot of possible skips occurred in the level which had to be fixed by adding more spikes and thus making the level harder. Some of these bugs were made into features since they couldn't be fixed. Additionally, a terminal velocity had to be added to prevent the player from sinking into platforms when they jumped.

6. Future Fixes and Updates

I would like to create a better collision system with the platforms since the current system isn't perfect and requires a specific level design to prevent bugs from occurring.
